# GitHub Copilot Instructions

## Project Overview

This is a full-stack web application using FastAPI (Python) for the backend and Nuxt 3 (TypeScript/Vue.js) for the frontend, containerized with Docker.

## Architecture

- **Backend**: FastAPI with SQLAlchemy ORM, PostgreSQL database, Alembic migrations
- **Frontend**: Nuxt 3 with TypeScript, Vue.js, Vuetify UI framework, Pinia for state management
- **Database**: PostgreSQL
- **Authentication**: JWT-based authentication with bcrypt password hashing
- **Development**: Docker Compose for local development, Poetry for Python dependency management

## Code Style & Standards

### Backend (Python/FastAPI)

- Use Python 3.12+ features
- Follow PEP 8 style guidelines (enforced by black, flake8, isort)
- Use type hints for all function parameters and return values
- Use Pydantic models for request/response schemas
- Follow SQLAlchemy 2.0+ async patterns
- Use dependency injection for database sessions and authentication
- Structure: models → schemas → CRUD → API endpoints
- Use structured logging and proper error handling
- Follow REST API conventions

### Frontend (TypeScript/Nuxt 3)

- Use TypeScript for all new code
- Follow Vue 3 Composition API patterns
- Use Pinia for state management
- Use Vuetify components for UI consistency
- Follow Nuxt 3 file-based routing conventions
- Use proper TypeScript interfaces for API responses
- Implement proper error handling and loading states
- Use composables for reusable logic

### Database

- Use Alembic for all database migrations
- Follow SQLAlchemy 2.0+ declarative model patterns
- Use proper foreign key relationships
- Include proper indexes for performance
- Use enum types where appropriate

## File Structure Patterns

- Backend: `/back/app/` with modular structure (models, schemas, crud, v1 API)
- Frontend: `/front/` with Nuxt 3 conventions (pages, components, stores, middleware)
- Shared: Docker configurations and environment files in `/secret/`

## Security Best Practices

- Always validate input data with Pydantic schemas
- Use JWT tokens for authentication
- Hash passwords with bcrypt
- Implement proper CORS settings
- Use environment variables for sensitive configuration
- Validate user permissions in API endpoints

## Development Workflow

- Use Poetry for Python dependency management
- Use Docker Compose for local development
- Follow pre-commit hooks for code quality
- Use Makefile commands for common tasks
- Write tests for critical functionality

## Common Patterns to Follow

1. **API Endpoints**: Use FastAPI dependency injection for auth and database
2. **Database Models**: Inherit from `Base` class, use proper relationships
3. **Frontend Pages**: Use layouts, middleware for auth, proper TypeScript
4. **State Management**: Use Pinia stores with proper TypeScript interfaces
5. **Error Handling**: Consistent error responses and user feedback

## Avoid

- Hardcoded configuration values (use environment variables)
- SQL injection vulnerabilities (use SQLAlchemy ORM)
- Missing type annotations in Python
- Any TypeScript usage in frontend
- Direct database access from API endpoints (use CRUD layer)
- Storing sensitive data in version control
